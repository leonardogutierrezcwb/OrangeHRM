🛡️ Segurança e autenticação
CT 005 - Acesso direto a páginas sem login

Tentar acessar /dashboard/index sem autenticação e validar se é redirecionado para /auth/login.

CT 006 - Sessão expirada

Simular o tempo de inatividade prolongada (via cy.wait() ou manipulação de cookies) e tentar continuar navegando.

👥 Administração (Admin)
CT 007 - Validação de campos obrigatórios ao cadastrar novo usuário

Tentar salvar sem preencher todos os campos e validar mensagens de erro.

CT 008 - Cadastro de usuário com username duplicado

Cadastrar um usuário com o mesmo username e validar a mensagem de erro.

👤 PIM (Employee Management)
CT 009 - Edição de funcionário

Após cadastrar, procurar pelo funcionário e editar informações (ex: sobrenome).

CT 010 - Excluir funcionário (se permitido)

Validar que o registro é removido e não aparece na listagem.

🔎 Recrutamento (Recruitment)
CT 011 - Validação de e-mail inválido

Testar campo de e-mail com dados malformados e verificar mensagens de erro.

📊 Dashboard e My Info
CT 012 - Verificar dados do Dashboard

Validar se os cards e gráficos estão sendo carregados corretamente após login.

CT 013 - Alteração de dados pessoais

Modificar dados na seção "My Info", como endereço ou telefone (se permitido).

📂 Documentação / Upload
CT 014 - Upload de documento no perfil (My Info ou Recruitment)

Validar se é possível fazer upload de arquivos (PDF, imagens) e verificar se aparecem na interface.

🔄 Logout e fluxo final
CT 015 - Logout via URL direta

Acessar diretamente a URL de logout (/auth/logout) e validar comportamento.

💡 Dicas extras:
Teste de responsividade (modo mobile/tablet) via .viewport().

Validações com interceptações de API via cy.intercept() para garantir que chamadas de rede estão funcionando corretamente.

Performance básica: validar se o carregamento das páginas está abaixo de um tempo aceitável com cy.clock() + cy.tick() para testes controlados.
